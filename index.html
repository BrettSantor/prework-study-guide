HTML

  • HTML is the structure or bones of the webpage
  • There are two main elements that make up the webpage known as parent elements
  • The <head> element contains information about the webpage
  • Inside the head element you can use metadata tags such as <title>, <style>, <base>, <link> and <meta>
  • The <body> element represents the visible content shown to the user
  • Inside the <body> element you can use tags such as <h1>-<h6>, <p>, <ul>, <ol> and <li>
  • Elements inside the <head> and <body< elements are known as child elements
  • Element refers to the entire element from opening tag to closing tag
  • tag refers only to what is inside the angle brackets < >
  • Elements start and stop with tags. An example would be <head>metadata goes here</head>
  • Semantic html elements are elements that are named after its content and role it plays on a webpage
  • Not all html are semantic but it is best practice to use semantic elements when possible

CSS

  • A margin indicates how much space we want around the outside of the element
  • A padding indicates how much space we want around the content inside an element
  • You can use inline CSS to directly to html file but must be edited individually when you make a change
  • Internal style sheet contains CSS rules in the head section of the html file and only apply to that page
  • External style sheet are the preferred method since keeping CSS rules in a seperate file makes design changes easier
  • To use an external style sheet you must <link> it in the html <head>
  • Every html element is in a box consisting of Margin, Border, Padding and Content. This is known as CSS box model

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: pulls the main branch from github and checks if you are up to date locally

Javascript

  • Control flow is the order in which a computer executes code in a script
  • A variable is a named container that allows us to store data in our code
  • Javascript is executed from the top down, order of code is important
  • Creating a function does not execute it, it must be called in our code
